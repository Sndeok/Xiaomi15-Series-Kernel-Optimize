# 更新日志

## v3.1

- 新增独立 `CHANGELOG.md` 更新日志文件。
- No-NTSync / With-NTSync 两个在线更新 JSON 的 `changelog` 均改为指向 `CHANGELOG.md`，不再显示项目 README。

## v3.0

- 模块描述新增内核模块加载状态显示。
- 同名 `.ko` 已存在时，先尝试卸载，再加载本模块内置 `.ko`。
- 卸载失败、载入失败、重新载入失败等状态会在模块描述中明确提示。
- With-NTSync 版本也会显示 `ntsync.ko` 加载状态。

## v2.9

- 调整内核模块加载逻辑：如果检测到同名模块已经存在，会先尝试 `rmmod` 卸载，再加载本模块内置 `.ko`。
- 如果同名模块卸载失败，为避免重复 `insmod` 失败，会跳过该模块并继续执行后续优化。

## v2.8

- 修复 Android 环境下 `grep "^$name\\b"` 不兼容导致模块描述无法识别已加载内核模块的问题。
- 改用 `awk` 精确匹配 `lsmod` 第一列。

## v2.6

- 刷入过程逐项提示的显示间隔从 0.2 秒调整为 0.28 秒。

## v2.1

- 刷入过程增加逐项完成提示。
- 开机后根据模块加载状态与日志开关状态更新模块描述。
- 新增 Magisk 在线更新 JSON。
